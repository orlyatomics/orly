- **Fixed**: `~TClient` could deadlock forever in `IoThread.join()` when a server push raced destruction: the dispatch thread's #503 exception exit skipped the internal-pipe close that was the i/o thread's only shutdown signal. The i/o thread now polls the destruction semaphore directly (and closes its pipe end on the way out, which also unblocks a dispatch thread parked mid-`Read()`), and the dispatch thread closes the pipe on its exception path too. A second interleaving of the same race — a pending push request's context ref making the dispatch thread the client's LAST owner, so `~TClient` self-joined on it (terminate; previously a silent two-thread leak) — is fixed by detaching instead and having the dispatch loop check a self-destruction flag before touching the object again. Root cause of the `import_replication.test` CI wedge: a client destroyed right after a replication-ack notification storm, under scheduler starvation, hit the race roughly every other run (#537).
