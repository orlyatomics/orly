- **Fixed**: `~TClient` could deadlock forever in `IoThread.join()` when a server push raced destruction: the dispatch thread's #503 exception exit skipped the internal-pipe close that was the i/o thread's only shutdown signal. The i/o thread now polls the destruction semaphore directly (and closes its pipe end on the way out, which also unblocks a dispatch thread parked mid-`Read()`), and the dispatch thread closes the pipe on its exception path too. This was the root cause of the `import_replication.test` CI wedge — a client destroyed right after a replication-ack notification storm, under scheduler starvation, hit the race roughly every other run (#537).
