- **A pre-restart POV id is now refused cleanly instead of silently resurrecting empty** (#439). The repo save side (`TManager::OnClose`) was never ported, so a POV's un-promoted updates cannot survive a restart — but the POV's *durable record* does, and `TPov::GetRepo` blindly re-created the backing repo, minting an empty shell parented to global whose reads fell through as if the lost writes never happened. A reloaded POV (stream-constructed from a durable disk layer) now only *attaches* to a live repo — which exists on a promoted slave, where replication built it — via the new `TryGetLiveRepo` (live-only lookup, never constructs, never touches the unported #173 reload path); if the repo is gone, the client gets a "povs are ephemeral" error telling it to create a new POV. Documented in PROTOCOL.md; `tests/restart_test.sh` now carries a pre-restart POV id across the restart and asserts the refusal.
